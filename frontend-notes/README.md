## 🎯 Time Allocation Strategy

For technical tests, focus effort where companies actually test:

| Topic       | Priority |
|-------------|----------|
| JavaScript  | 60% |
| CSS         | 30% |
| HTML        | 10% |

Reasoning: most companies focus heavily on JavaScript concepts, DOM manipulation, and problem-solving — not raw HTML memorization. Bootstrap sits on top of CSS knowledge and is usually assessed through practical tasks rather than direct MCQs.

## 🗓️ Suggested Study Order

1. **HTML** — Beginner → Advanced (fastest to complete, foundation for everything else)
2. **CSS** — Beginner → Advanced (spend real time on Flexbox/Grid — most-tested layout topic)
3. **JavaScript** — Beginner → Advanced (heaviest weight — allocate the most days here)
4. **Bootstrap** — Beginner → Advanced (fast once CSS fundamentals are solid)
5. **Project Notes** — apply everything in a real build (cart → checkout → payment)
6. **JavaScript/Practical/JS-Tricks.md** — final-day rapid revision before a test/interview

## ✅ Golden Rule for Tests
Answer *exactly* what's asked — don't over-explain. Interviewers value precision (e.g., "Difference between X and Y" answered in a clean table beats a paragraph).

## 📝 How to Use the Practice Questions
Each Beginner/Intermediate/Advanced file ends with a **Practice Questions** section. Recommended approach:
1. Read the section once.
2. Close the file and attempt to answer the questions from memory.
3. Check back against the notes for anything missed.
4. Revisit weak topics the next day (spaced repetition).
